## 1. ANALYSE PANORAMATA

### 1.1 Vue d'ensemble

**Panoramata** est une plateforme de benchmarking marketing full-funnel. Elle permet de surveiller automatiquement la stratégie digitale complète de n'importe quelle marque — emails, newsletters, publicités (Meta/TikTok/Google/LinkedIn/Pinterest/YouTube), landing pages, SMS, flows d'automatisation, tech stack.

**Cible :** Équipes marketing e-commerce, agences créatives, consultants Paid Media, CMOs — PME à grandes entreprises (L'Oréal, Pinterest, TikTok sont clients).

**Industries couvertes :** Mode, beauté, e-commerce, SaaS, B2B, finance, voyage.

**Base de données :** +100M assets marketing, 100k+ marques, 150 pays, 300 secteurs.

### 1.2 Fonctionnalités

| Module | Description |
|--------|-------------|
| Email tracking | Abonnement auto aux newsletters concurrentes |
| Ads spy | Meta, TikTok, Google Display, LinkedIn, Pinterest, YouTube |
| Landing pages | Capture + historique des LP concurrentes |
| SMS tracking | Flux SMS des marques suivies |
| Flows automation | Capture des séquences d'automatisation |
| SEO + Tech stack | Monitoring site web et technos utilisées |
| Benchmarks temps réel | KPIs vs concurrents + normes sectorielles |
| Agents IA | Insights personnalisés automatisés |
| Recherche avancée | +100M assets avec filtres exclusifs |
| Content calendar | Calendrier éditorial des concurrents |

### 1.3 Pricing

| Plan | Mensuel | Annuel | Limites |
|------|---------|--------|---------|
| Startup | €99/mois | €990/an | 20 concurrents, 6 mois historique, 3 users |
| Professional | €149/mois | €1 790/an | Concurrents illimités, historique illimité, 10 users |
| Advanced | €379/mois | €3 990/an | 20 users, 50 dashboards, CRM segmentation, A/B detection |
| Enterprise | Sur devis | Sur devis | Tout + account manager dédié |

### 1.4 Points forts (avis G2/Capterra, note ~4.9/5)

- **Gain de temps massif** : économie déclarée de 20h+/mois par utilisateur
- **Vue full-funnel** : seul outil qui connecte pub → landing page → email → SMS
- **Usage quotidien** : devenu source n°1 pour les consultants Paid Media
- **Facilité d'utilisation** : filtres intuitifs, partage de listes simple
- **Support réactif** : régulièrement cité comme point fort
- **Approche agence** : multi-clients avec dashboards partagés

### 1.5 Points faibles (avis & manques identifiés)

- **Prix élevé** : €99/mois juste pour 20 concurrents est restrictif pour les PME
- **Axé e-commerce / DTC** : peu adapté aux SaaS B2B ou industries hors retail
- **Profil G2 inactif** depuis +1 an → signal de négligence marketing
- **Pas de LLM visibility** : ne mesure pas la visibilité IA des marques
- **Pas de veille SEO deep** : tech stack oui, mais pas de tracking de positions
- **Pas de scoring** : donne des données brutes, peu de recommandations actionnables
- **Interface parfois surchargée** : trop d'informations en même temps
- **Données non segmentées par rôle** : marketer ≠ CEO ≠ agence

## 2. ANALYSE MINEA

### 2.1 Vue d'ensemble

**Minea** est un outil AdSpy spécialisé e-commerce/dropshipping. Son focus : identifier les produits gagnants et les créatifs publicitaires performants sur Facebook, TikTok et Pinterest. C'est l'outil de référence pour les dropshippers.

**Cible :** Dropshippers, e-commerçants Shopify, entrepreneurs cherchant des produits à lancer.

**Base de données :** ~900M publicités, 90 000 nouveaux ads/jour sur Meta + TikTok + Pinterest.

### 2.2 Fonctionnalités

| Module | Description |
|--------|-------------|
| Facebook Ads spy | Analyse campagnes par secteur, région, engagement |
| TikTok Ads spy | Contenu viral, tendances Gen Z/millenials |
| Pinterest Ads spy | Mode, déco, DIY — contenu visuel |
| Magic Search (IA) | Recherche produits gagnants par IA |
| Creative Finder (IA) | Trouver les créatifs qui performent |
| Success Radar | Top 100 produits mis à jour toutes les 8h (Business only) |
| Influencer module | Analyse des campagnes influenceurs |
| Shopify Store insights | Analyse des boutiques Shopify concurrentes |
| TikTok Shop insights | Données ventes TikTok Shop |
| Competitor Finder | Identification automatique des concurrents |

### 2.3 Pricing

| Plan | Mensuel | Annuel (~30% off) | Crédits |
|------|---------|-------------------|---------|
| Starter | $49/mois | $34/mois | 10 000 crédits |
| Premium | $99/mois | $69/mois | 100 000 crédits |
| Business | $399/mois | $299/mois | 150 000 crédits |

**Note :** TikTok + Pinterest = Premium minimum. Success Radar = Business uniquement.

### 2.4 Points forts (Trustpilot 3.9/5, reviews dropshipping)

- **Données massives** : 900M ads, mise à jour quotidienne
- **Spécialisation dropshipping** : ultime pour trouver des produits gagnants
- **Multi-plateformes** : seul à couvrir Facebook + TikTok + Pinterest ensemble
- **Success Radar** : top produits actualisés toutes les 8h (très apprécié)
- **IA intégrée** : Magic Search et Creative Finder performants
- **Prix d'entrée accessible** : $34/mois en annuel

### 2.5 Points faibles (Trustpilot, Reddit, reviews 2024/2025)

- **Système de crédits frustrant** : s'épuisent rapidement, force à upgrade
- **Problèmes de facturation/annulation** : plainte récurrente n°1 sur Trustpilot
- **Coaching overpriced** : $1000/mois perçu comme arnaque
- **Hors e-commerce = inutile** : apps, B2B, SaaS → pas adapté
- **Courbe d'apprentissage** : interface complexe pour les débutants
- **Données parfois datées** : mises à jour 3-5 jours pour certains segments
- **Focus trop produit** : peu de veille sur la marque globale (email, SEO, LLM)
- **Pas de multi-tenant** : difficile pour agences gérant plusieurs clients
- **Géo-focus occidental** : limité pour marchés non-anglophones

## 4. SPYMARK — POSITIONNEMENT & DIFFÉRENCIATION

### 4.1 Ce que font bien les concurrents et qu'on doit implémenter

**De Panoramata :**
1. **Benchmarks comparatifs** avec normes sectorielles → voir où on se situe vs marché
2. **Content calendar** de la concurrence → visualisation temporelle des actions marketing
3. **Dashboards partageables** → essentiel pour les agences et consultants
4. **Résumés hebdomadaires IA** → digest automatique envoyé par email
5. **Approche agence** (multi-workspace, facturation séparée par client)
6. **Landing page monitoring** → capturer les pages de destination des ads

**De Minea :**
1. **Success Radar / Trending alerts** → les publicités les plus actives du moment
2. **Filtres avancés** sur les ads (secteur, format, durée d'activité, engagement)
3. **Creative scoring** → noter les créatifs par performance estimée
4. **Données produit** quand pertinent pour e-commerce
5. **Onboarding guidé** avec use cases concrets

### 4.2 Ce qui est mal fait et qu'on peut faire mieux

**Failles de Panoramata à exploiter :**
- Prix trop élevé ($99 pour 20 concurrents seulement) → offrir plus de concurrents au même prix
- Pas de **LLM visibility** → notre différenciateur n°1
- Interface surchargée → UX épurée, focus sur l'essentiel
- Pas de **scoring actionnable** → transformer données en recommandations concrètes
- Profil G2 abandonné → investir le marketing produit

**Failles de Minea à exploiter :**
- Système de crédits frustrant → pricing transparent à l'usage ou flat
- Annulation difficile → offrir annulation instantanée (trust builder)
- Pas adapté au B2B → segment peu servi, strong opportunity
- Pas de vue marque globale → SpyMark couvre email + SMS + SEO + LLM + Ads

### 4.3 Positionnement SpyMark

> **"SpyMark est la seule plateforme de veille concurrentielle qui mesure la présence de vos concurrents partout où les décisions d'achat se prennent — des réseaux sociaux aux moteurs de recherche IA."**

**Différenciateurs uniques :**
1. **LLM Visibility** (ChatGPT, Perplexity, Gemini) — personne d'autre ne fait ça
2. **Veille full-stack** (Ads + Email + SMS + SEO + LLM + Social) sur une seule plateforme
3. **Pricing transparent** sans système de crédits
4. **Multi-tenant natif** avec Clerk Organizations
5. **Cible B2B élargie** : pas seulement e-commerce mais SaaS, fintech, retail

## 5. PLAN D'ACTION PRODUIT

### Phase 1 — Foundation (0-2 mois) : Finir le MVP

**Priorités immédiates :**

- [ ] **Alertes temps réel** : notification email/push quand un concurrent lance une nouvelle pub
- [ ] **Filtres avancés sur les ads** : par format, plateforme, première vue, statut actif/inactif
- [ ] **Dashboard benchmarks** : KPIs comparatifs (nb ads actives, fréquence, formats utilisés)
- [ ] **Export PDF/CSV** des rapports concurrentiels
- [ ] **Onboarding wizard** : 5 étapes guidées pour ajouter les premiers concurrents

### Phase 2 — Différenciation (2-4 mois)

- [ ] **LLM Visibility dashboard** : visualisation claire des scores par concurrent + tendance
- [ ] **Weekly digest email** : résumé automatique des mouvements concurrentiels
- [ ] **Content calendar view** : timeline des publications ads/emails par concurrent
- [ ] **Landing page capture** : screenshot + archivage des landing pages linked aux ads
- [ ] **Scoring IA des créatifs** : estimer la qualité/originalité d'un créatif concurrent

### Phase 3 — Growth (4-8 mois)

- [ ] **Mode Agence** : workspaces séparés, facturation par client, rapports white-label
- [ ] **API publique** : permettre aux équipes dev d'intégrer les données
- [ ] **Alerts intelligentes** : ML pour détecter patterns (ex : concurrent qui augmente fortement ses dépenses)
- [ ] **Benchmark sectoriel** : se comparer aux moyennes de son industrie (données anonymisées)
- [ ] **Chrome extension** : capturer des ads depuis la bibliothèque Meta ou TikTok

## 6. RECOMMANDATIONS HOSTING & BASE DE DONNÉES

### 6.1 Architecture actuelle

```
Next.js 14 App Router
├── Clerk (Auth)
├── Prisma ORM
├── PostgreSQL (Supabase)
├── Supabase Storage (screenshots)
└── Playwright (scraping)
```

### 6.2 Analyse de la situation actuelle

**Ce qui est bien :**
- Supabase = excellent choix pour phase MVP/early SaaS : auth-compatible, Storage intégré, Postgres natif
- Prisma = ORM solide, typage fort, migrations fiables
- L'architecture avec `DATABASE_URL` + `DIRECT_URL` est la bonne pratique pour Supabase + Prisma (connection pooling via PgBouncer)

**Risques identifiés :**
- Supabase gratuit → limites basses (500MB DB, 1GB storage, 2 CPU shared)
- Playwright scraping sur la même instance = dangereux en prod (consomme CPU/RAM)
- Pas de queue de jobs → scraping synchrone = timeouts serverless
- Pas de Redis → pas de cache, requêtes DB répétées inutilement
- Logs/monitoring absents → aucune visibilité en prod

### 6.3 Recommandations par stade

---

#### STADE 1 — MVP / Pre-Launch (maintenant → ~50 users)

**Hosting Frontend + Backend :** **Vercel Pro** ($20/mois)
- Zero-config Next.js, edge network, preview URLs par PR
- Idéal pour itérer vite
- Limite : fonctions serverless max 60s → suffisant pour le MVP

**Base de données :** **Supabase Pro** ($25/mois)
- 8GB DB, 100GB storage, dedicated compute
- Auth intégrée (même si vous utilisez Clerk, Storage reste utile)
- Connexion pooling PgBouncer inclus → essentiel pour serverless
- Migration 0 effort (vous êtes déjà sur Supabase)

**Scraping :** **Railway** ($5-15/mois)
- Séparer le service Playwright sur Railway (container persistant)
- Évite les timeouts Vercel sur les fonctions de scraping longues
- Peut tourner en worker background avec queue simple

**Coût total Stade 1 :** ~$50-60/mois

```
Vercel Pro        → Next.js frontend + API routes
Supabase Pro      → PostgreSQL + Storage screenshots
Railway Worker    → Playwright scraping service
```

---

#### STADE 2 — Early Traction (50-500 users)

**Hosting :** **Vercel Pro** (augmente usage automatiquement)

**Base de données :** **Supabase Pro** → migrer vers **Neon Scale ($69/mois)** si traffic variable
- Neon = serverless Postgres qui scale à zéro → 45ms p95 vs 80ms Supabase sur charge
- Branching natif = preview DB par PR (excellent pour CI/CD)
- Meilleur pour workloads intermittents (scraping par batch)
- **Garder Supabase Storage** séparé pour les screenshots (S3-compatible, pas cher)

**Queue de jobs :** **Upstash QStash** (~$10/mois) ou **Inngest** (gratuit jusqu'à 50k events)
- Évite les timeouts pour scraping asynchrone
- Retry automatique, monitoring des jobs
- Inngest s'intègre parfaitement avec Next.js App Router

**Cache :** **Upstash Redis** (~$10/mois)
- Cache les résultats de requêtes fréquentes (stats dashboard)
- Rate limiting pour l'API publique future

**Monitoring :** **Sentry** (gratuit jusqu'à 5k errors/mois) + **Vercel Analytics**

**Coût total Stade 2 :** ~$120-150/mois

```
Vercel Pro        → Next.js + API routes
Neon Scale        → PostgreSQL serverless (branching, auto-scale)
Supabase Storage  → Screenshots seulement (S3 bucket)
Railway           → Playwright worker + crons
Inngest           → Job queue (scraping, emails, LLM checks)
Upstash Redis     → Cache + rate limiting
Sentry            → Error monitoring
```

---

#### STADE 3 — Scale (500-5000 users)

**Hosting :** Évaluer **Railway full-stack** (tout sur Railway = $60-100/mois total au lieu de $300+ Vercel)
- Railway gère Docker, Next.js, workers, crons, DB, Redis sur une seule facture
- Pas de "Vercel Tax" sur les membres d'équipe
- Alternative : Fly.io si besoin de edge computing global

**Base de données :** **Neon Business ($700/mois)** ou **RDS PostgreSQL** (AWS)
- À ce stade, envisager read replicas pour les requêtes analytics lourdes
- Potentiellement séparer DB transactionnelle et DB analytics (TimescaleDB pour time series)

**Storage :** **Cloudflare R2** (0 egress fees, $0.015/GB stockage)
- Nettement moins cher que Supabase Storage à grande échelle
- Compatible S3 → drop-in replacement

**CDN :** **Cloudflare** (gratuit jusqu'à gros usage)
- Cache des screenshots/images publicitaires
- Protection DDoS incluse

**Coût total Stade 3 :** ~$300-600/mois selon usage

### 6.4 Plan de migration base de données

**Migration Supabase → Neon (quand vous êtes prêts) :**

```bash
# 1. Export Supabase
pg_dump $SUPABASE_DATABASE_URL > backup.sql

# 2. Créer projet Neon
# Dashboard Neon → New Project → Copy connection string

# 3. Import dans Neon
psql $NEON_DATABASE_URL < backup.sql

# 4. Mettre à jour .env
DATABASE_URL="postgresql://user:pass@ep-xxx.neon.tech/neondb?sslmode=require"
DIRECT_URL="postgresql://user:pass@ep-xxx.neon.tech/neondb?sslmode=require"

# 5. Tester avec prisma
npx prisma db pull
npx prisma generate
```

**Garder Supabase Storage** pour les screenshots (aucune raison de migrer, coût marginal).

### 6.6 Checklist avant mise en production

```
Infrastructure
□ Supabase Pro activé (pas gratuit)
□ DATABASE_URL avec pooler (port 6543, ?pgbouncer=true&connection_limit=1)
□ DIRECT_URL sans pooler (port 5432, pour migrations Prisma)
□ Vercel Pro activé
□ Variables d'environnement toutes renseignées sur Vercel
□ Domaine custom configuré

Scraping / Workers
□ Service Playwright séparé sur Railway
□ Queue Inngest configurée pour les jobs async
□ Crons Inngest : ads quotidien, SEO quotidien, LLM hebdo

Monitoring
□ Sentry installé (SENTRY_DSN dans .env)
□ Vercel Analytics activé
□ Alerts Sentry pour erreurs critiques

Sécurité
□ APP_SECRET fort et unique (min 32 chars)
□ CLERK_WEBHOOK_SECRET configuré et validé svix
□ Headers de sécurité dans next.config.js
□ Rate limiting sur /api/scraping routes
```

| | Panoramata | Minea | SpyMark (cible) |
|--|-----------|-------|-----------------|
| **Forces** | Full-funnel, fiable | Volume data, dropshipping | LLM unique + full-stack |
